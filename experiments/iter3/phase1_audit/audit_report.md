## 三方审计共识

### P0 必修 (影响策略根基)

#### 1. 中枢滑动窗口 → 需引入状态机
- **问题**: 每次新笔时取最近4点重算中枢，无延伸/升级/完成态
- **后果**: 中枢边界随新笔漂移、震荡市反复产生伪中枢、3B/3S信号重绘
- **三方一致**: GPT-5.2(P0)、Gemini(极高P0)、Claude(P0)
- **修复方向**: 
  - 中枢确立后固定zg/zd
  - 后续笔在区间内→延伸（中枢不变）
  - 笔完全脱离→离开事件
  - 离开后回抽不破→确认三买/三卖
  - 新的3笔重叠→新中枢

#### 2. 2B/2S "背驰" 逻辑方向反了
- **问题**: 代码判 `diff增强`（p_now.diff > p_prev.diff），实际是动量追踪
- **缠论标准**: 背驰 = 价格创新高/低但力度减弱（MACD面积缩小）
- **后果**: 2B变成追涨买入、2S变成跌得更猛才退出
- **三方一致**: GPT-5.2(P0)、Gemini(极高P0)、Claude(P0)
- **修复方向**: 
  - 改为比较两段同向走势的MACD面积（柱体积分）
  - 面积缩小 = 背驰 → 才触发信号
  - 或改名为"动能确认"保留当前逻辑但不叫背驰

### P1 重要

#### 3. 止损 ±1 硬编码
- GPT-5.2: P1(跨品种不适配)
- Gemini: P1(高波动品种被噪音扫出)
- 建议: 参数化为 ATR 比例或至少 pricetick 相关

#### 4. 3B/3S 缺少完整结构链
- GPT-5.2: P0(结构不等价)
- Gemini: P1(依赖中枢修复)
- 当前只检查"两个端点都在zg/zd外"，缺少"离开段→回抽段→不破确认"的完整链路
- 建议: 绑定中枢生命周期事件

#### 5. 15m MACD 过滤太弱
- 仅判 diff > dea，不等于大级别结构方向
- GPT-5.2: P1、Gemini: P2
- 建议: 增加15m笔方向判断或中枢状态

### P2 待优化

#### 6. 包含处理方向初始化
- `_inclusion_dir==0` 时默认=1（向上），可能早期/震荡时偏移
- GPT-5.2: P0、Gemini: P1
- 建议: 由最近两根非包含K线关系确定

#### 7. min_bi_gap 增强
- 仅用K线间隔，未加波幅门槛
- 建议: 双阈值 = K线间隔 + 最小ATR波幅
