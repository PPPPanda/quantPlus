# Iteration 12 Phase 2 — 失败模式分解

## 4个负合约诊断摘要

### p2401（最严重：-672.9pts 基线）
- **54笔交易**，WinRate 35.2%，**PF=0.75**（严重不足）
- AvgWin: 35.8pts vs AvgLoss: -26.1pts → 盈亏比仅 1.37:1
- **全月负收益**：2023-08到2023-12每月都是负的
- MaxConsecLoss: 9笔
- **核心特征：低波动阴跌**（见chan-work.md标注"死亡共振"）
  - 特征：ret ≈ maxdd，从头亏到尾
  - 平均单笔盈利极低（35.8pts），说明趋势幅度小、trailing几乎无法锁盈
  - 单笔最大亏损-72pts，不算极端，但亏损笔数太多

**失败模式**：
1. **过度交易**：54笔/81个交易日 = 0.67笔/天，在低波动环境中太频繁
2. **盈亏比不足**：趋势幅度小 → trailing无法激活或锁利微薄
3. **连亏频繁**：低波动中信号噪音大，连续假突破

### p2201（-355.6pts 基线）
- **56笔交易**，WinRate 33.9%，PF=1.03（勉强盈亏平衡）
- **2021-08月灾难性**：8笔全亏，-480pts（占总亏损的绝大部分）
- 2021-09、2021-11 盈利：分别 +318、+280pts

**失败模式**：

2. **恢复力强**：后期月份能盈利，说明不是系统性失效，而是初始化问题
3. **单笔最大亏损-174pts**：可能是止损过宽或市场跳空

## 跨合约共性失败模式

### FM-1: 合约初期连亏（p2201-08月, p2309-04月）
- **原因**：策略MACD/ATR指标在前几十根bar时计算不稳定（EMA收敛需要~30根bar）
- **对策候选**：
  - 加"预热期"：开盘后前N个5m bar不开仓（如前60根 = 5小时）
  - 或用更小窗口的ATR初始化

### FM-2: 低波动过度交易（p2401全程）
- **原因**：低波动中ATR小 → 止损近 → 频繁被扫 → 重新入场 → 循环亏损
- **对策候选**：
  - ATR波动率门槛：ATR < 某阈值时不开仓
  - 或ATR百分位过滤：当前ATR低于近N期ATR中位数时降频/不开仓

### FM-3: 超长连亏（p2305的12笔连亏）
- **原因**：当前断路器cb=6/60在极端连亏时仍不够
  - p2305的12笔连亏 → cb=6暂停60bar后重新开仓 → 继续亏
- **对策候选**：
  - 递进式断路器：第一次触发暂停60bar，第二次暂停120bar...
  - 或滚动窗口亏损上限：近N笔累亏超过阈值则更长暂停

### FM-4: 止损过紧/盈亏比不足（p2401的avgWin仅35.8）
- **原因**：buffer=atr*2%过小，在低波动中stop几乎贴着入场价
- **对策候选**：
  - 提高buffer：atr*5% 或固定最小buffer(如 2*pricetick=4pts)
  - 或基于中枢ZD的结构止损
