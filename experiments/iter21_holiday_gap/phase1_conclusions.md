# Phase 1 结论：节假日跳空对缠论分型的影响

## 数据分析结果

### 跳空统计（12个合约）
- **总跳空事件**: 2455
- **节假日跳空**: 1694 (69.0%)
- **时段跳空**: 485 (19.8%)
- **跳空延续率**: 49.7%（接近随机）

### 关键洞见
1. **跳空方向不可预测** - 延续率49.7%意味着跳空后的方向几乎是随机的
2. **平均跳空幅度约4x ATR** - 最大可达30x ATR
3. **节假日跳空是主要来源** - 占比69%

## GPT-5.2 缠论审计结论

### Q1：停盘后的K线是否形成有效分型？
**有效**。缠论分型用的是"相邻K线"的概念，本质是**在所选级别K线序列上的相邻**（index相邻），而不是日历上连续。

### Q2：跳空导致的价格不连续会让包含处理方向失效吗？
**不会从定义上失效**。包含处理依赖的是相邻K线高低点的集合关系，不要求价格路径连续。但跳空会显著增加"非包含"的概率，使方向更容易被迅速确认。

### Q3：108课是否有关于跳空的处理规定？
108课体系把缺口视为"走势的一部分/最低级别的连接方式"：
- **"缺口不等于中枢，只是连接中枢的最低级别……"**
- **"缺口表明离开原来中枢的力量比较大……"**

### Q4：跳空跨越中枢区间是否意味着中枢失效？
**不自动失效**。中枢的成立/延续/结束由笔序列及其重叠区间定义，跳空跨越中枢是"离开中枢"的极端表现，但中枢是否结束需要等"离开的一笔"完成确认。

## 新方案：桥接K线（Bridge Bar）

### 核心思想
把跳空显式建模为**最低级别走势连接段**，让序列在结构上连续，而不修改分型/包含/笔/中枢的定义。

### 实现方法
对每个session边界跳空，构造一根**虚拟K线**：
```python
bridge = {
    'open': prev_close,
    'close': cur_open,
    'high': max(prev_close, cur_open),
    'low': min(prev_close, cur_open),
    'volume': 0,
    'isSynthetic': True,
    'bridge': True,
}
```

### 关键规则
1. **Bridge Bar 参与相邻关系**（分型、包含处理）
2. **Bridge Bar 不参与笔最小长度计数**（用 `real_count` 而非 `index` 距离）
3. **跳空作为 feature 而非规则重置开关**（`leave_by_gap=True` 用于力度评估）

### 优势
- 不改任何缠论定义
- 分型不会因"突然断层"产生误判
- 包含方向不需要硬重置
- 中枢的"离开/回抽/扩展"可以更一致地追踪

---

## 与现有方案（S26/S27）的对比

| 方案 | S26（冷却） | S27（重置方向） | Bridge Bar |
|------|------------|----------------|------------|
| 核心逻辑 | 跳空后暂停交易 | 重置包含处理方向 | 把跳空建模为走势段 |
| 缠论依据 | 无（启发式） | 弱（假设方向污染） | 强（最低级别连接） |
| 改动范围 | 信号层过滤 | 结构层重置 | 数据预处理 |
| 副作用 | 可能错过好机会 | 可能破坏已有结构 | 笔计数需特殊处理 |

---

## 决策
采用 **Bridge Bar** 方案作为 Phase 4 的实现目标。
