# Iteration 13 Summary — p2401 做空可行性深度研究

## 研究目标
按 chan-work.md 大循环流程，专题研究：
1. 为什么 p2401 不能做空？
2. 有没有条件性做空方案能避免灾难？

## Phase 1: 三方审计（Claude + GPT-5.2 + Gemini）

### 三方一致结论
- **全面做空永不可行**：p2209 做多反弹赚 6939pts 是总收益支柱
- **3S 做空仅在"趋势性下跌+中枢递降+大周期确认"下有正期望**
- **p2401 高位做多全亏的根因**：5m信号在大级别空头中逆势开多（"级别陷阱"）

## 核心结论

### 为什么不能做空？（用数据说话）
1. **p2209 的盈利机制是"在极端下跌中买反弹"**（+6939pts）
2. **任何做空机制都会在反弹前开空、反弹时被止损**，直接干扰多头入场时机
3. **棕榈油下跌特征：急跌+V型反弹**，做空窗口极短
4. 即使用"3个中枢递降"这样的极端严格条件，p2209仍损失5778pts
5. p2401的改善（+113pts）远不足以弥补p2209的损失（-5778pts）

### p2401 为什么亏？
1. **不是因为不能做空**——即使在上涨月份（8月+640），策略也亏钱
2. **核心原因是"级别陷阱"**：5m信号在大级别空头中逆势开多
3. **主要亏损在7200-7500区间**（31笔-966pts），不是极端追高
4. **全局参数无法同时满足所有合约**——改善p2401必然伤害趋势合约

### 当前最优状态
- 基线参数（cb=7/70, div_threshold=0.39, mpa=3.0）：
  - TOTAL=13497.5pts
  - p2401=-672.9pts（需-180目标，缺口492.9pts）
  - p2305=-183.6pts（需-180目标，差3.6pts）
  - **p2401 是当前最大的"不可解"合约** — 在不伤其他合约的前提下，现有框架能力范围内没有找到让它达到-180的方案

## 后续建议
1. **接受 p2401 为"结构性亏损合约"**：低波动阴跌环境对只做多策略天然不利
2. **考虑合约级别的参数适配**（Per-contract parameters）：让 p2401 用不同的 cb/cooldown
3. **探索更高级别的过滤器**：引入日线级别中枢/趋势判断来过滤5m信号
4. **p2305 仅差3.6pts**：微调 mpa 或 div_threshold 可能足够
