## 1. 核心发现（颠覆性结论）

### 1.1 实际数据推翻"节后信号低质量"假设

| 指标 | 节后交易 | 普通交易 | 差异 |
|------|----------|----------|------|
| 交易数 | 9 | 347 | - |
| **胜率** | **66.7%** | 41.2% | **+25.5%** |
| 总盈亏 | -714.2 pts | -1214.6 pts | - |
| **平均盈亏** | **-79.4 pts** | -3.5 pts | **-75.9 pts** |

**结论**：节后信号胜率不低，甚至偏高；问题在于 **一旦亏损，单笔损失极大**。

### 1.2 灾难来源定位

| 日期 | 节日 | 合约 | 亏损 | 占比 |
|------|------|------|------|------|
| 2022-05-05 | 劳动节 | p2209 | -535.8 | 75% |
| 2022-05-05 | 劳动节 | p2209 | -501.8 | 70% |
| 2022-04-06 | 清明 | p2205 | -63.8 | 9% |

> **2022 劳动节后 p2209 的两笔交易贡献了 1037.6 pts 亏损，占节后总亏损的 145%**（因为其他节后交易有盈利抵消）

### 1.3 根因分析

1. **跳空不是信号失效，而是止损失效**
   - 节后开盘跳空平均 134 pts（普通 42 pts，3.2x）
   - 当跳空方向与持仓相反时，止损被远距离穿透
   - 固定 ATR 止损无法应对"开盘即大亏"场景

2. **节后第一根 bar 的 ATR 是滞后的**
   - 使用节前 ATR（未包含跳空幅度）
   - 实际波动 >>预设止损距离

## 2. 方案评估更新

### 2.1 不再推荐的方案

| 方案 | 原推荐理由 | 为何不推荐 |
|------|-----------|-----------|
| S25 缓冲期 | 避开节后假信号 | 数据显示节后胜率反而高 |
| S28 结构重置 | 消除历史污染 | 实现复杂，收益不明确 |

### 2.2 强烈推荐的方案

**S27 跳空后 ATR 自适应（首选）**

```python
# 伪代码
if is_first_bar_of_day():
    gap = abs(open_price - prev_close)
    if gap > 1.5 * prev_atr:
        # 临时放大 ATR 基准
        effective_atr = max(prev_atr * 1.5, gap / 2)
        atr_boost_bars = 6  # 持续 30 分钟
```

**理由**：
- 不阻止进场（保留节后大行情机会）
- 仅调整风控（止损距离动态适应跳空幅度）
- 预期减少单笔亏损幅度 50%+

### 2.3 可选辅助方案

**S26 极端跳空检测（安全网）**

```python
if gap > 3.0 * prev_atr:  # 仅极端跳空
    skip_signals_for_bars = 3  # 跳过 15 分钟
```

**理由**：
- 仅针对"开盘即崩"场景（如 2022 劳动节）
- 不影响正常节后交易
